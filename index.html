3. スプリントの流れ

スプリントとは、ITプロジェクトのアジャイル開発における専門用語で「一定の短い期間ごとに、作業の計画を立てて実現する方式」です。

CloudTech Academyでは目標とするAWS技術領域を12のスプリントに分割し、2週間で1つのスプリント学習の実施を基本的な流れとしています。

1. 動画学習

CloudTechの学習動画を視聴し、AWSの理解を深めます。CloudTechの動画はスプリントの単位でまとまっているので、上から順番に視聴していだければ問題ありません。

2. 課題

スプリントごとに課題を用意しています。

課題の構成

  1. 動画の理解度確認
    動画の理解度を確認するために、簡単な問題に回答していただきます。基本的に、動画で登場する用語について簡単に説明してもらう問題となります。
  2. 応用問題
    スプリントの内容を応用するための問題となります。AWS構成図を記載する問題であったり、実際のAWS設定値を考えてもらう問題などを用意しています。
  3. ハンズオン課題（疑似案件）
    CloudTech Academyの目玉でもある、疑似案件を想定したハンズオン課題に取り組んでいただきます。

ヒアリングシート

課題の回答をヒアリングシートに記載し、提出していただきます。ヒアリングシートの内容は、別途メンターからご提示いたします。

ヒアリングシートの提出後、メンターが確認を行い、フィードバックさせていただきます。メンターからのフィードバックを待つ間、次のスプリントに進んでいただいて構いません。

3. メンタリング

メンタリングチケットを使用してメンターとオンライン会議で課題に対する取り組みや学習状況をフィードバックします。

4. スプリントの期間について

目安としては1つのスプリントを2週間で完了させますが、理解度などに応じてメンターと調整の上で学習計画を組み立てることもできます。

必要に応じてメンターとご調整ください。

4. ハンズオン課題（擬似案件）

この課題では、クラウドエンジニアの実務を体験します。あなたのミッションは、擬似的な予約管理システムを稼働させるためのAWSクラウド環境を構築することです。

あなたはインフラチームの「メンバー」として、「リーダー」役のメンターの指示に基づき作業を進めます。リーダーから提供される要件をもとに、指定されたインフラ環境を構築し、期日までに成果物を提出してください。

1. 成果物のイメージ

最終的な成果物として以下のような構成を作成していきます。

テーマ 該当スプリント 概要
1. Webアプリケーションのデプロイ スプリント1〜5 擬似予約管理システムをデプロイし、動作環境を構築します。最終的にはコンテナで運用します。
2. CI/CDパイプライン スプリント6 アプリケーションの変更を検知し、自動でデプロイさせる環境を準備します。
3. インフラのコード化 スプリント7 インフラ環境をコード管理し、メンテナンス性を高めます。
4. サーバレスアーキテクチャ スプリント8〜9 サーバレス技術を使った口コミ投稿機能を構築します。
5. データ分析 スプリント11 システムで扱うデータの分析や可視化を行える環境を構築します。
6. 組織管理 スプリント12 組織でAWSを利用するためのマルチアカウント管理やセキュリティ管理の設定を行います。

2. 進め方

これらを一気に作るのではなく、スプリントごとに段階的に構築していきます。

テーマ 概要 利用サービス
スプリント1: ネットワークとサーバ 基本のWebサーバとAPIサーバをEC2インスタンス上に構築します。 Amazon VPC, Amazon EC2
スプリント2: 基本サービス2 RDSインスタンスを構築し、APIサーバから接続できるようにします。 Amazon RDS, AWS IAM
スプリント3: 冗長化構成 APIサーバを複数のAZに配置し、ALBによる負荷分散とオートスケーリングを設定します。 Application Load Balancer, Auto Scaling Group, CloudWatch
スプリント4: コンテンツ配信 WebサーバをS3に移行し、CloudFront、Route53、ACMでHTTPS化します。 Amazon S3, CloudFront, Route53, ACM
スプリント5: コンテナ APIサーバをECS（Fargate）に移行します。 Amazon ECS, AWS Fargate, Amazon ECR
スプリント6: DevOps CI/CDパイプラインを構築し、デプロイを効率化します。 AWS CodePipeline, CodeBuild, CodeDeploy
スプリント7: インフラのコード化 AWS構成をTerraformでコード化します。 Terraform
スプリント8: サーバレス サーバレスサービスを組み合わせ、口コミ投稿機能を構築します。 AWS Lambda, DynamoDB, API Gateway
スプリント9: 生成AIと応用 口コミへの回答を生成AIで作成し、メール返信機能を構築します。 Amazon Bedrock, Amazon SES
スプリント10: アプリケーション統合 サーバレス機能をSQSとStep Functionsで統合します。 Amazon SQS, AWS Step Functions
スプリント11: データ分析 Glueでデータを抽出し、AthenaとQuickSightで分析と可視化を行います。 AWS Glue, Amazon Athena, QuickSight
スプリント12: マルチアカウント マルチアカウント管理やセキュリティ設定を行います。 AWS Organizations

3. AWS利用料金について

この課題では無料利用枠だけではカバーできないため、一部料金が発生します。特に、サービスを使用し続けると料金が増加する可能性があるため、使わないリソースは停止してください。

  • 各スプリントのページに料金を抑えるための注意点を記載しています。
  • 適切に管理すれば、全体で約3,000円以内に抑えられる見込みです。

5. メンタリング

1. メンタリングとは

メンタリングは、Slackによるテキストコミュニケーションだけではカバーしきれない部分をサポートするために行います。以下を目的として、全12回のセッションを実施します。

  • 課題へのフィードバック
  • 学習内容の理解度チェック
  • 学習状況の確認とサポート
  • 不明点や不安の解消
  • 学習方法のアドバイス

これにより、効率的かつ効果的な学習を進めるお手伝いをいたします。

実施方法

原則として、Google Meetを使用してオンラインで行います。

実施タイミング

通常は各スプリント期間の最後に実施することを推奨します。ただし、ご都合に応じて調整可能です。担当メンターと日程を調整し、案内に沿ってセッションを実施してください。

2. キャンセルポリシー

メンタリングの振替

  • メンタリング日の 前日20:00まで にご連絡いただければ、別の日程への振替が可能です。
  • 振替を希望される場合は、Slackで担当メンターにお知らせください。

当日キャンセル

  • 前日20:00以降のキャンセルは 当日キャンセル とみなされます。この場合、メンタリングチケット1回分が消費されます。
  • 当日キャンセルでも、メンターは待機しておりますので必ずご連絡をお願いいたします。
  • 特別なペナルティはありませんのでご安心ください。

無断キャンセル

  • 当日に連絡なくメンタリングに参加されない場合、無断キャンセル とみなされます。この場合、振替はできず、チケット1回分が消費されます。
  • 正当な理由がない無断キャンセルが続くと、学習進行が難しくなる場合がありますのでご注意ください。

遅刻について

  • 開始時刻に遅れて参加された場合も、終了時刻は予定どおりとなります。延長対応はできませんのでご了承ください。

はい、可能です。担当メンターにご相談ください。

受講期限までに12回のメンタリングが実施できなかった場合、期間後に対応してもらえますか？

対応できません。受講期間内に完了していただく必要があります。ただし、有料の延長プランもご用意していますのでご相談ください。

2. 技術質問の範囲

対象となる技術質問

技術質問の対象は以下の内容です。

  1. 教材に対する不明点の確認
    例：動画のXXXの説明は、XXXという理解で合っていますか？
  2. ハンズオン課題の実装確認やエラー解決
    例1：課題のXXX部分をこのように実装しようと思いますが、懸念点はありますか？
    例2：XXXというエラーが発生しており、解決策を教えてほしい。

対象外の技術質問

以下は対象外となります。

  1. 実業務に関連する質問
    例：業務で発生したEC2エラーを解析してほしい。
  2. カリキュラム外の質問
    例：Azureで同じ構成を構築する方法は？
  3. 企業に関する質問
    例：XXX企業から内定が出ましたが、入社すべきですか？

Q&A

なぜテンプレートに沿って質問する必要があるのですか？

テンプレートにより効率的な質問ができ、メンターも適切な回答がしやすくなります。また、仮説を立てる力が身につき、実務でも評価されるエンジニアに成長できます。

メンターの回答が24時間かかる間に学習が止まってしまいます。

エラー箇所以外を進めるか、次のスプリントに着手してください。

ヒントしか提示されず、答えを教えてもらえませんでした。

CloudTech Academyでは、自立的な問題解決スキルを重視しているため、ヒントを提供します。再質問は遠慮なく行ってください。

初歩的なことを聞いて怒られませんか？

ご安心ください。理解度には個人差がありますので、遠慮なくご質問ください。
